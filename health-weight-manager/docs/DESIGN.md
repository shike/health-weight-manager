## 1. 项目概述

`health-weight-manager` 是一个运行在 OpenClaw 上的 Skill，用于帮助用户进行每日体重记录、周期性体重回顾以及定时健康提醒。它通过 OpenClaw 的 `cron` 工具实现自动化提醒，通过文件系统持久化存储体重数据，并通过消息渠道（Telegram/Slack/WhatsApp 等）与用户交互。

### 1.1 核心价值

- **低门槛记录**: 用户只需回复数字即可记录体重，无需打开 App。
- **自动化提醒**: 早晨称重提醒、全天健康提醒、每周回顾全部自动触发。
- **数据可视化**: 自动生成体重变化趋势，让用户直观看到进步。
- **隐私优先**: 所有数据存储在本地文件，不上传云端。

## 2. 功能需求

### 2.1 核心功能

| 功能编号 | 功能名称 | 描述 | 触发方式 |
|---------|---------|------|---------|
| F01 | 早晨体重提醒 | 每天早上固定时间提醒用户记录体重 | cron 定时任务 |
| F02 | 体重记录 | 接收用户输入的体重数字并存储 | 用户主动回复 / 指令 |
| F03 | 体重查询 | 查看最近 N 天的体重记录 | 用户主动指令 |
| F04 | 体重趋势分析 | 计算周/月平均值、变化量 | 用户主动指令 |
| F05 | 每周体重回顾 | 每周固定时间生成体重周报 | cron 定时任务 |
| F06 | 每日健康提醒 | 每天固定时间发送健康小贴士 | cron 定时任务 |
| F07 | 目标体重管理 | 设置目标体重，跟踪完成进度 | 用户主动指令 |
| F08 | 数据导出 | 将体重数据导出为 CSV 格式 | 用户主动指令 |

### 2.2 提醒场景详细设计

#### 场景 1: 早晨体重提醒 (F01)

- **触发时间**: 每天 07:00（可配置）
- **消息内容**:
  ```
  🌅 早上好！该记录今天的体重啦～
  请直接回复你的体重（单位：kg），例如：65.4
  ```
- **行为**: 发送提醒消息后，等待用户在当前会话中回复。
- **异常处理**: 如果用户当天已记录，则跳过提醒。

#### 场景 2: 每周体重回顾 (F05)

- **触发时间**: 每周日 20:00（可配置）
- **消息内容**:
  ```
  📊 本周体重回顾
  ━━━━━━━━━━━━━━━━━
  本周平均: 65.2 kg
  上周平均: 65.8 kg
  变化: -0.6 kg ⬇️
  
  最高: 65.5 kg (周一)
  最低: 64.9 kg (周五)
  
  记录天数: 5/7
  目标体重: 62.0 kg (还剩 3.2 kg)
  ━━━━━━━━━━━━━━━━━
  继续保持！💪
  ```

#### 场景 3: 每日健康提醒 (F06)

- **触发时间**: 每天 10:00、15:00、20:00（可配置）
- **消息内容**: 从健康小贴士库中随机选择一条发送
- **示例内容**:
  - 💧 记得每小时喝一杯水，促进新陈代谢！
  - 🚶 久坐后起来走动 5 分钟，活动一下筋骨。
  - 🥗 晚餐尽量清淡，控制碳水摄入有助于体重管理。

## 3. 数据模型

### 3.1 数据存储结构

所有数据存储在 `~/.openclaw/workspace/skills/health-weight-manager/data/` 目录下：

```
data/
├── weight_records.jsonl   # 体重记录（JSON Lines 格式）
├── user_config.json       # 用户配置
└── health_tips.json       # 健康小贴士库
```

### 3.2 体重记录格式 (weight_records.jsonl)

```json
{"timestamp": "2026-04-26T07:15:00+08:00", "weight": 65.4, "note": "空腹"}
{"timestamp": "2026-04-25T07:20:00+08:00", "weight": 65.6, "note": ""}
```

字段说明：
- `timestamp`: ISO 8601 格式的时间戳
- `weight`: 体重数值（kg，保留 1 位小数）
- `note`: 可选备注（如"空腹"、"饭后"等）

### 3.3 用户配置格式 (user_config.json)

```json
{
  "target_weight": 62.0,
  "reminder_morning": "07:00",
  "reminder_evening": "20:00",
  "weekly_review_day": "Sunday",
  "weekly_review_time": "20:00",
  "health_reminders": ["10:00", "15:00", "20:00"],
  "timezone": "Asia/Shanghai"
}
```

## 4. 指令集设计

用户可以通过以下指令与 Skill 交互：

| 指令 | 别名 | 功能 |
|-----|------|------|
| `记录体重 <数值>` | `体重 <数值>` | 记录一条体重数据 |
| `查看体重 [天数]` | `体重记录 [天数]` | 查看最近 N 天记录（默认 7 天） |
| `体重趋势 [周数]` | `趋势 [周数]` | 查看体重趋势分析（默认 4 周） |
| `设置目标 <数值>` | `目标体重 <数值>` | 设置目标体重 |
| `导出数据` | `导出` | 导出 CSV 格式的体重数据 |
| `健康提醒开关 [on/off]` | — | 开启/关闭健康提醒 |

## 6. Cron Job 配置

OpenClaw 的 `cron` 工具用于设置定时任务。本 Skill 需要以下 cron jobs：

| Job ID | 描述 | Cron 表达式 | 对应功能 |
|--------|------|-----------|---------|
| `hwm-morning` | 早晨体重提醒 | `0 7 * * *` | F01 |
| `hwm-health-1` | 上午健康提醒 | `0 10 * * *` | F06 |
| `hwm-health-2` | 下午健康提醒 | `0 15 * * *` | F06 |
| `hwm-health-3` | 晚间健康提醒 | `0 20 * * *` | F06 |
| `hwm-weekly` | 每周体重回顾 | `0 20 * * 0` | F05 |

> 注：cron 表达式遵循标准 Unix cron 格式。OpenClaw 会在触发时调用 skill 中定义的对应处理流程。

## 7. 安全与隐私

### 7.1 数据安全

- 所有数据存储在本地文件系统，不依赖外部 API。
- 数据文件权限设置为 `600`（仅所有者可读写）。
- 不收集任何个人身份信息（PII）。

### 7.2 Skill 权限声明

```yaml
tools:
  - cron          # 用于定时提醒
  - message       # 用于发送消息到用户渠道
  - read          # 用于读取数据文件
  - write         # 用于写入数据文件
  - bash          # 用于执行数据处理脚本
```
